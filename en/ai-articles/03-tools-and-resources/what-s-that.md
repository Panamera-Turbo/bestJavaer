# Who do I think it suits?

If you're just one person writing code and you're mostly sitting in front of a computer, just use Claude Code/ Codex.

Slock may be a little heavy for you.

But if you're already using al agent, and you're thinking about it all the time:

```text
Can this mission let a Agent run?
This report?
Can the context of this project be left to a certain Agent for a long time?
Can I just call it work from the web/phone without opening the terminal?
```

Then Slock is worth trying.

It's not about how to talk to AI, it's about how to work with a group of AI Agents.

That's what it says on the front page:

> Where humans and AI agents build together.

Not using AI as an input box, but as a collaborator who can be assigned tasks, can participate in discussions and can be permanently present.

Of course, it looks a little early at this point, and UI has a character and a little ahead of product patterns.

But the direction I'm accepted.

Because when the AI programming tool is getting stronger, the question may not be whether the individual Agent will write code, but:

```text
How do I manage more than one?
How do I know what they're doing?
How do I make them not repeat the same thing?
How do I put humans, code stores, missions, discussions, AI implementation in one place?
```

Slock wants to do this thing.

Brief summary:

```text
Claude Code / Codex: A strong AI programmer
Slock: an office where many AI programmers work with humans.
```

If you use it as an alternative to Claude Code, it might not be necessary.

But if you think of it as "Ai Age Slack," it suddenly makes sense.
