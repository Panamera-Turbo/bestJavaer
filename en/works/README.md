# Works & Open Source

Here's the work I've done, opened up, or maintained. They're not all big, but they all come from the pains of real use: publishing, AI coding workflow, volume statistics, interactive experiments, and the content station itself.

## Products and works

- [Layweout](https://github.com/crisxuan/layweout)

Online address:[https://layweout.vercel.app](https://layweout.vercel.app)

Markdown/HTML layout desk for the public launch process. It does not focus on previews that look fancy, but on the steady reproduction of technical articles, AI articles into the public domain editor to minimize patterns of confusion and collapse.

## Open Source Project

- [Agent Workflow Kit](https://github.com/crisxuan/agent-workflow-kit)

A workflow tool kit to help project determine "Whether or not to access Agent workflow, what to receive". It removes the process selection, risk judgement, certification orders and collaboration complexity from AI coding, allowing projects to be assessed before being regulated.

Related articles:[Connecting Agent Workflow Kit to Your Project](../ai-articles/01-agent-and-coding/connecting-agent-workflow-kit-to-your-project.md)

- [Codex Usage Estimator](https://github.com/crisxuan/codex-usage)

Local estimation tool for Codex usage, token costs and subscriptions. It dissects "how the amount fell so quickly" into observable data, allowing easy access to the contained input, output, task grouping and overall consumption.

Related articles:[I finally pulled out the Codex's volume. It wasn't the output.](../ai-articles/01-agent-and-coding/i-finally-pulled-out-the-codex-s-volume-it-wasn-t-the-output.md)

This site itself. The old Java, co-opt, JVM, operating systems, networks and database content continue to be archived, with new main lines moving to AI tool experience, Agent workflow, open-source project observation and personal experimental records.
