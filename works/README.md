# 我的作品 & 开源项目

这里放我自己做过、开源过，或者正在持续维护的作品。它们不一定都很大，但都来自真实使用里的痛点：写作发布、AI coding 工作流、用量统计、交互实验，以及这个内容站本身。

## 产品与作品

- [Layweout](https://github.com/crisxuan/layweout)

  在线地址：[https://layweout.vercel.app](https://layweout.vercel.app)

  面向公众号发布流程的 Markdown/HTML 排版工作台。它重点解决的不是“预览看起来花哨”，而是把技术文章、AI 文章稳定复制到公众号编辑器里，尽量减少样式错乱和导出崩坏。

  相关文章：[Layweout 公众号排版工作台](../ai-articles/03-tools-and-resources/Layweout%20%E5%85%AC%E4%BC%97%E5%8F%B7%E6%8E%92%E7%89%88%E5%B7%A5%E4%BD%9C%E5%8F%B0.md)

## 开源项目

- [Agent Workflow Kit](https://github.com/crisxuan/agent-workflow-kit)

  一套帮助项目判断“要不要接入 Agent workflow、该接到什么程度”的工作流工具包。它把 AI coding 里的流程选择、风险判断、验证命令和协作复杂度拆出来，让项目先评估，再上规约。

  相关文章：[Agent Workflow Kit 接入你的项目](../ai-articles/01-agent-and-coding/Agent%20Workflow%20Kit%20%E6%8E%A5%E5%85%A5%E4%BD%A0%E7%9A%84%E9%A1%B9%E7%9B%AE.md)

- [Codex Usage Estimator](https://github.com/crisxuan/codex-usage)

  面向 Codex 用量、token 成本和订阅额度的本地估算工具。它把“额度怎么掉得这么快”拆成可观察的数据，方便查看 cached input、output、任务分组和整体消耗。

  相关文章：[我终于把 Codex 的用量扒出来了，原来最贵的不是输出](../ai-articles/01-agent-and-coding/%E6%88%91%E7%BB%88%E4%BA%8E%E6%8A%8A%20Codex%20%E7%9A%84%E7%94%A8%E9%87%8F%E6%89%92%E5%87%BA%E6%9D%A5%E4%BA%86%EF%BC%8C%E5%8E%9F%E6%9D%A5%E6%9C%80%E8%B4%B5%E7%9A%84%E4%B8%8D%E6%98%AF%E8%BE%93%E5%87%BA.md)

- [bestJavaer / cxuan-ai-labs](https://github.com/crisxuan/bestJavaer)

  这个站点本身。旧的 Java、并发、JVM、操作系统、网络和数据库内容继续归档保留，新主线转向 AI 工具体验、Agent 工作流、开源项目观察和个人实验记录。

  站点入口：[cxuan-ai-labs](../README.md)
