# 今天阿里宣布全面封禁 Claude Code 了。

[English](../../en/ai-articles/04-industry-and-business/alibaba-company-wide-ban-on-claude-code.md) | [中文](./%E4%BB%8A%E5%A4%A9%E9%98%BF%E9%87%8C%E5%AE%A3%E5%B8%83%E5%85%A8%E9%9D%A2%E5%B0%81%E7%A6%81%20Claude%20Code%20%E4%BA%86%E3%80%82.md)

> 日期：2026-07-04

昨天阿里宣布全面封禁 Claude Code 了。

自 7 月 10 日起全面禁止内部员工在办公环境下使用该工具，并推荐使用自研的 Qoder 作为替代方案了。

这次封禁的原因，我想大家应该都知道了，就是由于 Claude Code 被曝存在植入后门的安全风险，而且精准的锁定了147个中国域名，美团、网易、百度、阿里、字节跳动等国内主流科技公司及 AI 实验室均在监控名单之上。

我这周写的几篇文章，完整的复盘了一下。

[Claude Code 为了封杀中国用户，竟然在代码里下毒。](https://mp.weixin.qq.com/s/rjyvfgTJ45e7nGcCGINlCQ)

[A 社封号惹众怒了，连大 V 都忍不了开喷了。](https://mp.weixin.qq.com/s/_a4LKBZbLQop_JqQqt4FiQ)

我觉得阿里这个做法，很有魄力。

这次阿里敢实现全面禁止 Claude Code，也让我对 Qoder 和 Qwen 大模型有了一些关注和期待，并且我也希望见到阿里在 AI 领域能有所突破。

阿里对员工不管是从报销额度还是提供免费模型额度的使用上来说，都很慷慨，不过当然这也是作为头部大厂和互联网公司的一大优势了。

我之前看有互联网公司的人发 token 消耗截图，接 API 的话，有的人夸张到一周能消耗几万美金，这还是 Opus 4.6 的时候了，如果换算成 Fable 5 ，那不敢想。

其实 A 社天天这么针对我们，会倒逼出国人一条弯道超车的魄力。

上一次这么跟我们搞的时候，是搞技术封锁，我们不得已做信创。那时候芯片被卡、操作系统被卡，硬是倒逼出了一条国产化的路。

今天 AI 又来玩这一套，软件层面给你埋后门、打标签、搞歧视性封号。

历史总是惊人地相似。

被卡过脖子的滋味不好受，但每一次被卡，也都在倒逼着我们进步。

其实国模已经不差了，这两天我看到了一项 GLM-5.2 和 Fable 5 的一个测评，有人用这俩模型做了仿真的 3D 地球对比，能看出来，模型能力差距已经拉的很小。

但就像是之前 DeepSeek 发布 V4 的时候说过的一句话：不诱于誉，不恐于诽，率道而行，端然正己。

芯片如此，操作系统如此，AI 大模型，也应当是如此。

我希望这次能倒逼着国模进步，不仅仅是阿里。

让大家吸上便宜好用的 token。
