虽然 GPT 5.6 正式发布了，但 Codex 却迎来了跟大家说再见的时候。

<img src="https://cdn.jsdelivr.net/gh/doggaifan/picbed/image-20260710062050003.png" alt="image-20260710062050003" style="zoom: 33%;" />

OpenAI 正式推出了 **ChatGPT Codex 和 ChatGPT Work**。

现在的 Codex ，叫 **ChatGPT Codex**。

这两个模式可以直接在 Codex 里面无缝切换。

<img src="https://cdn.jsdelivr.net/gh/doggaifan/picbed/image-20260710044232157.png" alt="image-20260710044232157" style="zoom: 67%;" />

ChatGPT  Work 的上线，我感觉有可能也是 OpenAI 要杀入办公场景的一个入口了。

不同于 Codex For developers 会展示思考过程，ChatGPT Work 把技术细节隐藏掉，主要面向非开发者。

<img src="https://cdn.jsdelivr.net/gh/doggaifan/picbed/image-20260710044141262.png" alt="image-20260710044141262" style="zoom:50%;" />

ChatGPT Work 是这次最值得关注的新功能。它不再只是回答问题，还能跨应用执行任务：连接你的 Google Drive、Slack、邮箱等工具，自己拆解复杂项目，分步骤独立完成，需要时才问你。

比如你给它一句话“把客户调研整理成营销方案”，它能自己读资料、写方案、生成不同市场的素材版本，全程带着上下文。

这也是之前说的，ChatGPT 和 Codex 可能要整合在一起了。

Work 也会登陆网页和手机端，意味着你不需要在电脑前也能发起和跟踪任务了。

很多产品比如 Qoder 、Trae 、WorkBuddy 国内的产品，其实都有两套，一个是面向程序员的、一个是面向职场人的。

只不过这次 OpenAI 把它们集成在一块了。

<img src="https://cdn.jsdelivr.net/gh/doggaifan/picbed/image-20260710045138967.png" alt="image-20260710045138967" style="zoom:50%;" />

这次浏览器也迎来重大更新。

在 Codex 内使用 Computer use 功能更丝滑了，而且支持已认证网站、多个标签页和文件下载。除此之外你的标签也能保留下来，更丝滑了。

<img src="https://cdn.jsdelivr.net/gh/doggaifan/picbed/image-20260710045405067.png" alt="image-20260710045405067" style="zoom:50%;" />

这个 Computer use 功能，能让你的 ChatGPT 使用任何 app ，越来越像人来使用。最关键的，使用速度更快了，而且更省 token 。

之前的 Computer use 功能我日常使用频率也很高，确实感觉效率有些慢了。

除此之外，GPT 5.6 在前端、可视化等方面也有十足的改进。OpenAI 这次也是放了一个不同类型生成的前端效果图。

![image-20260710051820738](https://cdn.jsdelivr.net/gh/doggaifan/picbed/image-20260710051820738.png)

这次更新，给大家还是带来了不少新东西，我觉得这些小细节和特性，能够我兴奋一段时间了。

GPT 5.6 三箭齐发，固然大家会陷入到一种爽感的氛围中，兴许听不得一些反对意见。

但我可能得跟大家说点不一样的了，我可能要泼点冷水了。

先放一张我最喜欢的测评机构的图。

![image-20260710072336557](https://cdn.jsdelivr.net/gh/doggaifan/picbed/image-20260710072336557.png)

这张图其实在说：Fable 5 和 GPT‑5.6 的胜负，要分综合能力和编码 Agent 两种方式来看。

- 上半图是综合 Intelligence Index（9 个不同测试混合）：Fable 5 60，GPT‑5.6 Sol 59，Terra 55。
  也就是说，综合分上 Fable 略高于 Sol，Terra 又低一档。
- 下半图是 Coding Agent Index（DeepSWE、Terminal-Bench、SWE-Atlas-QnA 的总和）：
  Sol 80，Terra 77，Fable 5 77。
  也就是说，做代码 Agent 时 Sol 第一；Terra 和 Fable 基本打平。

所以它不能得出 GPT‑5.6 全面吊打 Fable，反而说明：

Sol 是代码 Agent 这一条赛道的第一；
Fable 在综合能力上仍稍领先；
Terra 的定位很像编码 Agent ，成绩接近 Fable，但整体能力低一档的中档模型。

另外，max 是最高推理档，with fallback 表示 Fable 的安全回退配置也在运行；所以这不是纯模型裸体对决。不同工具链、推理档位和评测题集，都会改变排名。

而 Simon Willison 在他的博客提到了，GPT 5.6  。。。 可能效果没有 Fable 5 要强。

Simon Willison 有幸是提前能够体验的那一批人，所以有了足够的测试时间，他把前因后果说明白了。

<img src="https://cdn.jsdelivr.net/gh/doggaifan/picbed/image-20260710064142109.png" alt="image-20260710064142109" style="zoom:50%;" />

他说他们设计了一个 “智能体终极考试” ，旨在让每一个 token 都发挥更大的作用。

这项在这项测试涵盖了 55 个长时间运行的专业工作流评估，GPT-5.6 Sol 创下了 53.6 分的新高，比 Claude Fable 5（采用自适应推理模式）高出 13.1 分。

及时在 medium 模式下，它也以约四分之一的预估成本，领先 Fable 5 达 11.4 分。

这对实现更普及、更经济的智能至关重要：GPT-5.6 Terra 和 GPT-5.6 Luna 均优于 Fable 5，而成本仅为其十六分之一左右。

但是，在大家都很关注的 SWE-Bench 领域中， Fable 5 显著胜出 GPT-5.6 ，Fable 5 取得了 80% 的成绩，而 GPT-5.6 Sol 为 64.6%。

<img src="https://cdn.jsdelivr.net/gh/doggaifan/picbed/image-20260710064937746.png" alt="image-20260710064937746" style="zoom:50%;" />

当然，OpenAI 隔天就专门发了一篇文章，说 SWE-Bench Pro 大约 30% 的题目本身就有问题，可能存在题目描述不全、测试过严、答案不唯一这些情况，不能把这张成绩单当成最终裁判。

<img src="https://cdn.jsdelivr.net/gh/doggaifan/picbed/image-20260710064759981.png" alt="image-20260710064759981" style="zoom:50%;" />

他说就目前他的使用而言， A 社模型执行的复杂编码任务而言，它的表现并没有比 Fable 更好。

他针对 GPT 5.6 不同的版本，展示了 18 种不同的“鹈鹕”（pelican）配置——涵盖了三种不同模型在不同推理强度下的表现。还列出了它们的 Token 消耗及计算成本：成本最低的是 `gpt-5.6-luna`（推理强度为 None ），费用为 0.71 美分；成本最高的是 `gpt-5.6-sol`（推理强度为 Max ），费用为 48.55 美分。

<img src="https://cdn.jsdelivr.net/gh/doggaifan/picbed/gpt-5.6-pelicans-full.png" alt="GPT-5.6 Luna、Terra、Sol 在六档推理强度下生成鹈鹕骑车 SVG 的完整对比" style="zoom:50%;" />

*图片来源：Simon Willison，三列依次为 Luna、Terra、Sol，六行依次为 none 到 max。*
